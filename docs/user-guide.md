# Руководство пользователя системы DN Quest

## Введение

DN Quest — это современная платформа для проведения квестов и соревнований по программированию. Система предоставляет инструменты для создания, управления и участия в квестах, а также отслеживания прогресса и достижений.

---

## Быстрый старт

### 1. Запуск системы

Система состоит из нескольких микросервисов, работающих в Docker-контейнерах.

```bash
# Клонирование репозитория
git clone <repository-url>
cd dn-quest

# Запуск всех сервисов
docker-compose up -d

# Проверка статуса сервисов
docker-compose ps
```

### 2. Доступ к системе

После запуска система будет доступна по следующим адресам:

- **Frontend (Веб-интерфейс)**: `http://localhost:3000`
- **API Gateway**: `http://localhost:8080`
- **Swagger API Docs**: `http://localhost:8080/swagger-ui.html`
- **Grafana (Метрики)**: `http://localhost:3001` (admin/admin)
- **Jaeger (Трассировка)**: `http://localhost:16686`

### 3. Первый запуск

1. Откройте веб-интерфейс: `http://localhost:3000`
2. Зарегистрируйтесь или войдите в систему
3. Создайте или присоединитесь к квесту
4. Начните решать задачи!

## Регистрация и вход

### Регистрация нового пользователя

1. Перейдите на страницу регистрации
2. Заполните форму:
   - **Email**: Ваш адрес электронной почты (будет использоваться для входа)
   - **Имя**: Ваше имя или никнейм
   - **Пароль**: Минимум 8 символов, должен содержать:
     - Заглавные буквы
     - Строчные буквы
     - Цифры
     - Специальные символы
   - **Подтверждение пароля**: Повторите пароль

3. Нажмите кнопку "Зарегистрироваться"

После успешной регистрации:
- Вы автоматически войдете в систему
- На ваш email будет отправлено письмо с подтверждением
- Вам будет присвоена роль **USER**

### Вход в систему

1. Перейдите на страницу входа
2. Введите ваши учетные данные:
   - **Email** или **Имя пользователя**
   - **Пароль**

3. Нажмите кнопку "Войти"

### Восстановление пароля

Если вы забыли пароль:

1. Нажмите ссылку "Забыли пароль?" на странице входа
2. Введите ваш email
3. На email будет отправлена ссылка для сброса пароля
4. Перейдите по ссылке и введите новый пароль
5. Новый пароль будет активен немедленно

### Смена пароля

Войдите в систему и перейдите в настройки профиля:

1. Нажмите на аватар профиля → "Настройки"
2. Перейдите в раздел "Безопасность"
3. Введите текущий пароль
4. Введите новый пароль (дважды)
5. Нажмите "Сохранить изменения"

## Использование основных функций

### Создание квеста

1. Авторизуйтесь в системе
2. Перейдите в раздел "Квесты" → "Создать квест"
3. Заполните информацию о квесте:
   - **Название**: Имя квеста
   - **Описание**: Подробное описание задачи
   - **Тип квеста**:
     - *Индивидуальный* — решает один участник
     - *Командный* — решает команда
   - **Уровень сложности**: Легкий, Средний, Сложный, Эксперт
   - **Ограничение по времени**: Максимальное время на выполнение
   - **Начальные файлы**: Загрузите стартовые файлы (опционально)

4. Нажмите "Создать"

### Участие в квесте

1. Перейдите в раздел "Доступные квесты"
2. Выберите интересующий квест
3. Нажмите "Присоединиться"

Для командных квестов:
- Создайте команду или присоединитесь к существующей
- Пригласите участников по email
- Дождитесь подтверждения от приглашенных

### Решение задач

1. Откройте квест
2. Прочитайте описание задачи
3. Напишите код в встроенном редакторе
4. Запустите тесты для проверки решения
5. Отправьте решение на проверку

### Система оценивания

Решения оцениваются автоматически:
- **Успешно** — все тесты пройдены
- **Неудачно** — некоторые тесты не пройдены
- **Ошибка выполнения** — код вызвал исключение

Вы можете отправлять решения многократно, система сохраняет историю попыток.

### Лидерборд

Отслеживайте свои достижения и результаты других участников:

- **Место в общем рейтинге**
- **Количество пройденных квестов**
- **Среднее время выполнения**
- **Успешность решений**

### Уведомления

Система отправляет уведомления о:
- Приглашениях в команды
- Обновлениях квестов
- Результатах проверок
- Новых сообщениях

Управлять уведомлениями можно в настройках профиля.

## Архитектура системы

### Общая структура

Система построена по принципам микросервисной архитектуры:

```

   Frontend    (Vue 3 + Vite)
   Порт: 3000 

         
         

  API Gateway  (Spring Cloud Gateway)
   Порт: 8080 

         
    
                              
  
 Auth   File   Game   Queue  
Service Storage Logic  (Kafka)
  
```

### Микросервисы

#### 1. Authentication Service (Сервис аутентификации)
- **Порт**: 8081
- **Ответственность**: Регистрация, авторизация, управление пользователями
- **Технологии**: Spring Boot, Spring Security, JWT
- **База данных**: PostgreSQL

#### 2. API Gateway (Шлюз API)
- **Порт**: 8080
- **Ответственность**: Маршрутизация запросов, фильтрация, безопасность
- **Технологии**: Spring Cloud Gateway, Resilience4j
- **Функции**:
  - Аутентификация и авторизация
  - Балансировка нагрузки
  - Circuit Breaker
  - Rate Limiting

#### 3. File Storage Service (Сервис хранения файлов)
- **Порт**: 8082
- **Ответственность**: Загрузка, хранение и раздача файлов
- **Технологии**: Spring Boot, MinIO
- **Особенности**: Поддержка версионирования, контроль доступа

#### 4. Game Engine (Игровой движок)
- **Порт**: 8083
- **Ответственность**: Выполнение кода, проверка решений
- **Технологии**: Spring Boot, Docker (песочница)
- **Особенности**: Изолированное выполнение, ограничение ресурсов

### Брокер сообщений

**Apache Kafka** используется для асинхронной коммуникации:
- События пользователей (регистрация, обновление профиля)
- События квестов (создание, обновление, завершение)
- События решений (отправка, проверка, завершение)
- Уведомления

### Базы данных

- **PostgreSQL**: Основная реляционная БД
- **Redis**: Кэширование сессий и данных
- **Elasticsearch**: Поиск и аналитика

### Мониторинг и логирование

- **Prometheus + Grafana**: Метрики и дашборды
- **Jaeger**: Распределенная трассировка
- **ELK Stack** (Elasticsearch, Logstash, Kibana): Централизованное логирование

## API Endpoints

### Authentication Service

#### Регистрация
```http
POST /api/v1/auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "name": "Имя пользователя",
  "password": "SecurePass123!",
  "confirmPassword": "SecurePass123!"
}
```

#### Вход
```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "SecurePass123!"
}
```

#### Обновление профиля
```http
PUT /api/v1/users/{id}
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "Новое имя",
  "email": "new@example.com"
}
```

#### Смена пароля
```http
POST /api/v1/users/{id}/change-password
Authorization: Bearer <token>
Content-Type: application/json

{
  "currentPassword": "OldPass123!",
  "newPassword": "NewPass456!",
  "confirmPassword": "NewPass456!"
}
```

### Quest Service

#### Получение списка квестов
```http
GET /api/v1/quests?page=0&size=10&sort=createdAt,desc
Authorization: Bearer <token>
```

#### Создание квеста
```http
POST /api/v1/quests
Authorization: Bearer <token>
Content-Type: application/json

{
  "title": "Название квеста",
  "description": "Описание задачи",
  "type": "INDIVIDUAL",
  "difficulty": "MEDIUM",
  "timeLimit": 3600
}
```

#### Получение квеста
```http
GET /api/v1/quests/{id}
Authorization: Bearer <token>
```

#### Присоединение к квесту
```http
POST /api/v1/quests/{id}/join
Authorization: Bearer <token>
```

#### Отправка решения
```http
POST /api/v1/quests/{id}/submit
Authorization: Bearer <token>
Content-Type: application/json

{
  "code": "print('Hello, World!')",
  "language": "PYTHON"
}
```

### Team Service

#### Создание команды
```http
POST /api/v1/teams
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "Название команды",
  "questId": 1
}
```

#### Приглашение участника
```http
POST /api/v1/teams/{id}/invite
Authorization: Bearer <token>
Content-Type: application/json

{
  "email": "user@example.com"
}
```

### File Storage Service

#### Загрузка файла
```http
POST /api/v1/files/upload
Authorization: Bearer <token>
Content-Type: multipart/form-data

{
  "file": <binary_file>,
  "questId": 1
}
```

#### Скачивание файла
```http
GET /api/v1/files/{id}/download
Authorization: Bearer <token>
```
