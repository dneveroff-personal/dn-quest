# DN Quest - Микросервисная архитектура с Docker

## Обзор

DN Quest реализован как микросервисная архитектура с использованием Docker и Docker Compose для оркестрации контейнеров. Каждый сервис работает в изолированном контейнере с собственной базой данных и конфигурацией.

## Микросервисы

### 1. API Gateway
- **Порт**: 8080
- **Технологии**: Spring Cloud Gateway
- **Функции**:
  - Маршрутизация запросов к микросервисам
  - Аутентификация и авторизация
  - Rate limiting
  - CORS
  - Логирование запросов

### 2. Authentication Service
- **Порт**: 8081
- **База данных**: PostgreSQL (dnquest_auth)
- **Технологии**: Spring Security, JWT
- **Функции**:
  - Регистрация и аутентификация пользователей
  - Управление токенами
  - Права доступа и роли
  - OAuth2 интеграция

### 3. User Management Service
- **Порт**: 8082
- **База данных**: PostgreSQL (dnquest_users)
- **Функции**:
  - Управление профилями пользователей
  - Настройки пользователя
  - Социальные функции
  - История активности

### 4. Quest Management Service
- **Порт**: 8083
- **База данных**: PostgreSQL (dnquest_quests)
- **Функции**:
  - Создание и редактирование квестов
  - Управление уровнями
  - Медиа контент
  - Валидация квестов

### 5. Game Engine Service
- **Порт**: 8084
- **База данных**: PostgreSQL (dnquest_game)
- **Функции**:
  - Игровая логика
  - Обработка ответов
  - Прогресс игроков
  - Подсказки и коды

### 6. Team Management Service
- **Порт**: 8085
- **База данных**: PostgreSQL (dnquest_teams)
- **Функции**:
  - Создание и управление командами
  - Приглашения в команды
  - Роли в команде
  - Статистика команд

### 7. Notification Service
- **Порт**: 8086
- **База данных**: PostgreSQL (dnquest_notifications)
- **Технологии**: Kafka, Email, Telegram Bot
- **Функции**:
  - Email уведомления
  - Telegram бот
  - Push уведомления
  - Шаблоны уведомлений

### 8. Statistics Service
- **Порт**: 8087
- **База данных**: PostgreSQL (dnquest_statistics)
- **Функции**:
  - Сбор статистики
  - Аналитика
  - Отчеты
  - Лидерборды

### 9. File Storage Service
- **Порт**: 8088
- **База данных**: PostgreSQL (dnquest_files)
- **Технологии**: MinIO S3
- **Функции**:
  - Загрузка файлов
  - Обработка изображений
  - CDN интеграция
  - Метаданные файлов

### 10. Frontend
- **Порт**: 3000
- **Технологии**: Vue.js, Vite, Tailwind CSS
- **Функции**:
  - Пользовательский интерфейс
  - PWA поддержка
  - Реактивные обновления
  - Офлайн режим

## Инфраструктурные компоненты

### PostgreSQL
- **Порты**: 5432-5439 (для разных сервисов)
- **Функции**: Основное хранилище данных
- **Особенности**: Отдельная база для каждого сервиса

### Redis
- **Порт**: 6379
- **Функции**: 
  - Кэширование сессий
  - Временные данные
  - Rate limiting
  - Pub/Sub

### Kafka
- **Порт**: 9092
- **Функции**:
  - Асинхронная коммуникация
  - Событийная архитектура
  - Очереди сообщений
  - Stream processing

### MinIO
- **Порт**: 9000 (API), 9001 (Console)
- **Функции**: S3-совместимое хранилище файлов
- **Особенности**: Локальная альтернатива AWS S3

### Nginx
- **Порт**: 80 (HTTP), 443 (HTTPS)
- **Функции**:
  - Load balancing
  - Reverse proxy
  - SSL termination
  - Static files

## Мониторинг и логирование

### Prometheus
- **Порт**: 9090
- **Функции**: Сбор метрик со всех сервисов
- **Метрики**: JVM, HTTP, бизнес-метрики

### Grafana
- **Порт**: 3001
- **Функции**: Визуализация метрик
- **Дашборды**: Системные, бизнесовые, производительность

### Jaeger
- **Порт**: 16686
- **Функции**: Распределенный трейсинг
- **Отслеживание**: Запросы между сервисами

### ELK Stack
- **Elasticsearch**: 9200 - Хранение логов
- **Kibana**: 5601 - Визуализация логов
- **Logstash**: 5044 - Обработка логов

## Docker конфигурация

### Структура файлов

```
dn-quest/
├── docker-compose.yml              # Основная конфигурация
├── docker-compose.dev.yml          # Разработка
├── docker-compose.prod.yml         # Продакшен
├── docker-compose.full.yml         # Полный стек
├── docker/
│   ├── nginx/
│   │   └── conf.d/
│   │       └── default.conf        # Nginx конфигурация
│   ├── prometheus/
│   │   └── prometheus.yml          # Prometheus конфигурация
│   ├── grafana/
│   │   ├── provisioning/
│   │   │   ├── datasources/
│   │   │   └── dashboards/
│   │   └── grafana.ini             # Grafana конфигурация
│   └── jaeger/
│       └── jaeger-config.yml       # Jaeger конфигурация
├── scripts/
│   ├── start-all.sh                # Запуск всех сервисов
│   ├── stop-all.sh                 # Остановка всех сервисов
│   ├── restart-service.sh          # Перезапуск сервиса
│   ├── logs.sh                     # Просмотр логов
│   └── status.sh                   # Проверка статуса
├── .env.example                    # Пример конфигурации
├── .env.development                # Конфигурация разработки
├── .env.production                 # Конфигурация продакшена
├── .env.full                       # Конфигурация полного стека
└── .env.testing                    # Конфигурация тестирования
```

### Сети

- **dn-quest-network**: Основная сеть для сервисов
- **dn-quest-monitoring**: Сеть для мониторинга
- **dn-quest-frontend**: Сеть для frontend

### Volume'ы

- **postgres_data**: Данные PostgreSQL
- **redis_data**: Данные Redis
- **kafka_data**: Данные Kafka
- **minio_data**: Файлы MinIO
- **prometheus_data**: Метрики Prometheus
- **grafana_data**: Дашборды Grafana
- **elasticsearch_data**: Логи Elasticsearch

## Траблшутинг

### Общие проблемы
1. **Порты заняты** - Изменить в .env файлах
2. **Недостаточно памяти** - Увеличить Docker limits
3. **База данных не запускается** - Проверить volume permissions
4. **Сервисы не видят друг друга** - Проверить сети

### Диагностика
```bash
./dn-quest.sh status -h -d
./dn-quest.sh logs
docker-compose ps
docker network ls
docker volume ls
```

### Восстановление
